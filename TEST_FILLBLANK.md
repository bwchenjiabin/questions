## 可能的原因

### 1. 浏览器缓存
浏览器可能缓存了旧的组件状态。

**解决方法：**
- 硬刷新浏览器：Ctrl + Shift + R (Windows) 或 Cmd + Shift + R (Mac)
- 清除浏览器缓存
- 使用无痕模式测试

### 2. 开发服务器未热更新
Vite 开发服务器可能没有正确热更新。

**解决方法：**
```bash
# 停止开发服务器 (Ctrl + C)
# 重新启动
npm run dev
```

### 3. 已有数据的影响
如果页面加载时使用了已保存的题目数据，那些数据可能包含旧的空值。

**解决方法：**
- 创建新题目（而不是编辑已有题目）
- 切换到其他题型再切换回填空题

## 测试步骤

### 测试 1：创建新题目
1. 打开题目录入页面
2. 选择"填空"题型
3. 检查"答案"区域
4. **期望结果：** 只显示"+ 添加一个空"按钮，没有任何空值输入框

### 测试 2：切换题型
1. 先选择"单选"题型
2. 再切换到"填空"题型
3. 检查"答案"区域
4. **期望结果：** 只显示"+ 添加一个空"按钮，没有任何空值输入框

### 测试 3：添加空值
1. 选择"填空"题型
2. 点击"+ 添加一个空"按钮
3. **期望结果：** 显示"第一空答案"输入框
4. 再次点击"+ 添加一个空"
5. **期望结果：** 显示"第二空答案"输入框

## 调试建议

如果问题仍然存在，请检查：

1. **浏览器控制台**
   - 打开开发者工具 (F12)
   - 查看 Console 是否有错误
   - 查看 Network 标签，确认文件已更新

2. **Vue DevTools**
   - 安装 Vue DevTools 扩展
   - 检查组件的 data 属性
   - 查看 `fillBlankAnswer.blanks` 的值

3. **文件时间戳**
   - 确认文件确实已保存
   - 检查文件修改时间
